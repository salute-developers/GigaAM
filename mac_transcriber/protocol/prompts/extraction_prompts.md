# Промпты пайплайна протоколов

Три LLM-фазы. Полноту проверяет код (protocol_pipeline.py), промпты
лишь делают её достижимой: маленькие окна, жёсткий формат, запрет
на редактуру в фазе извлечения.

---

## Фаза 1. Extraction (вызывается на каждый chunk-NN.json)

Температура 0. На вход подаётся JSON чанка целиком.

```
Ты экстрактор фактов из транскрипта рабочей встречи. Ты НЕ редактор
и НЕ суммаризатор: твоя задача учесть каждый сегмент, ничего не сжимая.

Вход: JSON с сегментами {id, t0, t1, speaker, text}.

Верни строго JSON без пояснений:
{
  "classification": [{"id": "...", "class": "substantive|supporting|noise"}],
  "facts": [{
    "id": "F-<chunk>-<NN>",
    "type": "decision|task|question|risk|info",
    "text": "одно предложение, конкретика сохранена",
    "segments": ["S...."],
    "speaker": "кто сказал",
    "owner": "ответственный или null",
    "due": "срок или null",
    "entities": ["имена, системы, числа, термины из сегментов"]
  }]
}

Правила классификации:
- В classification обязан попасть КАЖДЫЙ id из входа, ровно один раз.
- substantive: решения, поручения, дедлайны, вопросы, риски, цифры,
  имена, названия систем, определения терминов, договорённости,
  несогласия, отложенные темы.
- supporting: подтверждения и уточнения без новой информации
  («ага», «да-да», «окей», приветствия, проверка звука).
- noise: обрывки, междометия, технические артефакты ASR.
- Сомневаешься между supporting и substantive: ставь substantive.

Правила фактов:
- Каждый substantive-сегмент привязан хотя бы к одному факту.
- Факт атомарен: одно решение, одна задача, один вопрос. Не склеивай.
- Дословно переноси: имена, числа, даты, сроки, ответственных,
  названия систем и таблиц, термины. Их потеря = ошибка.
- Не оценивай важность. Не выбрасывай «мелочи». Отбор будет позже,
  и он обратим, твоё извлечение - нет.
- Очевидные ошибки ASR в известных терминах нормализуй и сохрани
  оригинал в entities: «Niwisper» -> Whisper (ориг.: «Niwisper»).
- Реплика без контекста в начале чанка: класс supporting, в text
  факта пометь «контекст в предыдущем чанке».
```

## Фаза 2. Merge (один вызов, на вход все facts из всех чанков)

```
Ты сводишь факты, извлечённые из перекрывающихся фрагментов одной
встречи. Вход: список facts.

Верни строго JSON:
{
  "facts": [... объединённый список ...],
  "merged": {"F-03-04": "дубликат F-02-11", ...}
}

Правила:
- Удалять факт можно ТОЛЬКО как дубликат, с записью в merged.
  Любое другое исчезновение факта - ошибка.
- При слиянии дубликатов объединяй segments и entities обоих.
- Противоречащие факты не сливай: оставь оба, добавь обоим
  пометку "conflict": true.
- Формулировки можно улучшать, конкретику терять нельзя.
```

## Фаза 3. Protocol (один вызов: merged facts -> структура протокола)

```
Собери протокол встречи из фактов. Вход: facts после merge.

Верни строго JSON со структурой DATA из render_protocol.py
(meeting, tldr, decisions, tasks, questions, risks, timeline),
где каждый элемент дополнительно несёт "facts": ["F-..."].

Правила:
- Каждый факт типа decision/task/question/risk обязан попасть
  хотя бы в один пункт протокола либо в "merged" с причиной.
  Это проверит автомат, недостающие вернутся тебе же.
- Пункт протокола может объединять несколько фактов одной темы,
  перечисли их все в "facts".
- info-факты раскладывай по блокам timeline.
- ref пункта = первый сегмент первого факта + его таймкод.
- Решение - то, после чего кто-то действует иначе. Пересказ
  обсуждения решением не является: это info для timeline.
- Owner из фактов переносится в tasks как есть; нет owner -
  поле null, шаблон подсветит «не назначен».
```

## Фаза 1b. Gap recheck (если validate нашёл дыры)

```
Эти сегменты не были классифицированы или не привязаны к фактам.
Классифицируй каждый и создай недостающие факты по правилам Фазы 1.
Вход: gaps.json.
```

---

## Порядок запуска

```bash
python3 protocol_pipeline.py inventory transcript.md -o segments.json
python3 protocol_pipeline.py chunks segments.json --size 60 --overlap 6 -o chunks/
# LLM Фаза 1 по каждому chunk-NN.json -> склеить в extraction.json
python3 protocol_pipeline.py validate segments.json extraction.json
#   exit 2 -> gaps.json -> LLM Фаза 1b -> дописать в extraction.json -> validate снова
# LLM Фаза 2 -> facts_merged.json, Фаза 3 -> protocol.json
python3 protocol_pipeline.py gate extraction.json protocol.json -o coverage.json
#   exit 2 -> список сирот вернуть в Фазу 3 -> gate снова
python3 render_protocol.py   # protocol.json -> PDF
```

coverage.json кладётся рядом с PDF, в сам PDF не попадает.
